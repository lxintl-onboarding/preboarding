# 의사결정 기록

이 문서는 "왜 이렇게 만들었는가"를 남깁니다. 되돌리기 전에 읽어주세요.

## 1. 왜 단일 HTML 파일인가

앱 출시 없이 URL만 전달하는 것이 요구사항입니다. 정적 파일이면
GitHub Pages / Netlify 무료 티어로 배포할 수 있고, 인사팀이 직접
문구를 수정하기도 쉽습니다. 빌드 도구를 붙이면 이 장점이 사라집니다.

## 2. 왜 Google Apps Script인가 (연동 완료)

배송지 폼을 저장하려면 서버가 필요하지만, 클라이언트에 인증키를
넣을 수 없습니다. GAS는 구글 계정만으로 무료 서버리스 엔드포인트를
제공하고, 결과가 Google Sheets에 행으로 쌓여 인사팀이 바로 봅니다.

`GAS_ENDPOINT` 변수가 비어 있으면 "미리보기 모드"로 동작합니다 (현재는 채워져
연동된 상태 — 새 입사자 추가 절차는 `README.md` 참고).

**미해결**: 개인정보(주소·연락처)를 외부 무료 도구에 저장하는 것이
사내 정보보안 정책에 맞는지 IT팀 확인이 필요합니다.

## 3. 진행률 UI를 왜 세그먼트 바 + 분수 표기로 바꿨나

uniON 프리보딩 UX 케이스 스터디(Anfisa Novikova, Medium)에서
사용자들이 진행률 도식을 "폼 진행률"로 오해한 사례가 보고됐습니다.

퍼센트만 보면 "57%"가 무엇의 57%인지 즉각 읽히지 않습니다.
"4/7 완료"가 더 빠르게 이해됩니다. 세그먼트 바는 남은 칸을
셀 수 있어 다음 행동이 자명해집니다.

## 4. 왜 NEXT 뱃지로 하나만 강조하나

같은 케이스 스터디에서, 초기엔 상위 3개 태스크를 모두 펼쳐뒀다가
"혼란스럽다"는 피드백을 받고 현재 태스크 하나만 펼치도록 수정했습니다.
"지금 이것 하나"를 지목하는 것이 핵심입니다.

완료 항목 접기 토글도 같은 맥락입니다. 화면 소음을 줄이면
남은 일이 선명해집니다.

## 5. 왜 카드 스타일을 둘로 나눴나

같은 케이스 스터디의 테스터가 "왜 인터랙티브 요소와 비인터랙티브
요소가 같은 배경에 있는가"라고 지적했습니다.

- 누를 수 있는 것: 흰 카드 + 그림자 + 테두리 (`.card`)
- 읽기만 하는 것: 그림자 제거 + 웜톤 배경 + Gold 좌측 보더 (`.card-info`)

## 6. 왜 구비서류를 하위 체크리스트로 중첩했나

입사 안내 메일의 구비서류가 9종입니다. 기존엔 "제출 서류 확인"
한 줄이었는데, 실제로는 9개를 각각 발급받아야 합니다.

별도 섹션으로 분리하면 진행률이 2개가 되어 "명확한 진행률" 원칙이
흐려집니다. 그래서 상위 To-Do 1개 안에 9개를 접이식으로 넣고,
9개를 다 체크하면 상위가 자동 완료되게 했습니다.

이 과정에서 `label` 중첩이 발생해 상위를 `div`로 바꾸고
클릭·키보드 핸들러를 직접 붙였습니다.

## 7. 왜 Daum 우편번호 서비스인가

국내 웹 주소 검색의 사실상 표준입니다. 무료, API 키 불필요,
스크립트 태그 한 줄로 동작하며 팝업 UI까지 완성품으로 제공됩니다.

행안부 공식 API는 승인키 발급과 UI 직접 구현이 필요해 과합니다.

**주의**: 외부 CDN(`t1.daumcdn.net`)에서 스크립트를 로드합니다.
사내망 차단 여부 확인이 필요합니다.

## 8. 헤더 A안/B안이 왜 둘 다 남아있나

원본 로고는 LX Red 심볼 + LX Gray 워드마크입니다.
이를 Red 배경에 올리면 심볼 대비 1:1, 워드마크 1.26:1로 사라집니다.

- **A안**: 아이보리 배경 → 원본 로고 그대로 (CI 규정 100% 준수)
- **B안**: Red 배경 유지 + 화이트 반전 로고 (대비 11.93:1)

디자인만 보면 B안이 낫지만, **사내 CI에 화이트 반전 버전이
규정되어 있는지 확인**이 필요합니다. 현재 `logo-white.png`는
이미지 처리로 생성한 임시본입니다.

확정 시 다른 한쪽을 삭제하세요.

## 9. 왜 Figma MCP를 안 쓰나

html.to.design 등으로 HTML을 Figma로 가져올 수는 있지만 단방향
스냅샷입니다. Figma에서 수정해도 코드에 반영되지 않고, 역변환하면
기존 JS 로직이 깨집니다.

HTML을 원본(source of truth)으로 두고, Figma는 이해관계자 시안
논의용으로만 쓰는 것이 현재 판단입니다.
