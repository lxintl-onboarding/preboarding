Architecture Decision Record

LX 온보딩 앱 — 배송지 자동 연동, 왜 이 방식으로 결정했는가

입사자 맞춤 URL 발급부터 배송지 정보의 자동 시트 반영까지, 신규 비용·신규 계정 도입을 최소화하면서 담당자 교체·다수 대상자 처리에도 끊기지 않는 구조로 GitHub Pages와 Google Apps Script 조합을 채택합니다.

01 — 배경

무엇을 해결해야 했는가

현재 앱은 입사자 이름·출근일이 하드코딩된 프로토타입입니다. 실제 운영을 위해서는 다음 조건을 동시에 만족해야 했습니다.

담당자 비의존
특정 담당자 개인 계정·PC에 묶이지 않고, 담당자가 바뀌거나 여러 명이어도 그대로 운영 가능해야 함
인원 수 무관 자동화
1명이든 여러 명이든 동일한 절차로 URL 발급·발송·수합이 처리돼야 함
기기 간 상태 유지
입사자가 배송지를 제출한 뒤 다른 기기로 재접속해도 제출 여부가 그대로 보여야 함
신규 비용 최소화
별도 유료 SaaS 도입 없이, 이미 있는 도구에 가깝게 구현
02 — 결정

GitHub Pages(파일 배포) + Google Apps Script(데이터 처리)

두 서비스는 완전히 독립적으로 동작하며, 입사자의 브라우저가 페이지 로드 후 직접 GAS를 호출하는 것으로만 연결됩니다. 각각 "화면"과 "데이터"라는 역할을 정확히 나눠 맡습니다.

03 — 검토한 대안

왜 다른 조합이 아닌가

호스팅과 백엔드 두 축에서 각각 대안을 비교했습니다. 강조된 행이 채택안입니다.

05 — 전체 흐름

입사자 정보 입력부터 시트 반영까지

  1. 1
    Google Sheets에 입사자 정보 입력
    이름·출근일 등을 담당자가 행으로 추가 (Excel과 동일한 사용 경험)
  2. 2
    개인화 URL 자동 생성
    행 추가 즉시 수식으로 index.html#t=고유토큰 형태 링크가 만들어짐
  3. 3
    안내 메일 발송 현재 단계 선택사항
    회사 공식 메일 계정으로 발송 (GAS 전용 계정이 아님 — 06 참고)
  4. 4
    입사자가 링크 접속
    GitHub Pages가 서빙, GAS가 토큰으로 이름·출근일을 조회해 화면에 반영
  5. 5
    체크리스트 확인 · 배송지 입력 후 저장
    제출 여부가 GAS에 기록되어, 다른 기기로 재접속해도 “이미 제출됨”으로 표시
  6. 6
    Sheets에 실시간 반영
    담당자는 별도 작업 없이 Sheets에서 바로 확인, 필요 시 엑셀로 다운로드
06 — 계정 운영 방식

전용 계정 1개, 로그인은 관리자만

계정 하나를 여러 명이 로그인해서 함께 쓰는 방식이 아니라, 로그인 자체는 관리자 1명만 하고 나머지 담당자는 문서 공유로 참여하는 구조입니다. Google 계정 정책(“계정은 한 사람이 사용”이 원칙)과 충돌하지 않으면서도 여러 담당자가 실질적으로 함께 작업할 수 있습니다.

Google Account
전용 계정 1개
로그인은 관리자 1명만
(GAS 배포 등 드문 작업 전용)
Sheet 편집자 공유
담당자 A본인 개인 계정으로 접근
담당자 B본인 개인 계정으로 접근
담당자 C본인 개인 계정으로 접근

※ 평소 업무(입사자 추가, 제출 현황 확인)는 각자 자기 계정으로 하고, 전용 계정 로그인은 스크립트 재배포 등 드문 관리 작업에만 사용합니다. 담당자 교체 시에도 Sheet 공유 목록만 갱신하면 됩니다.

가입 유형 선택은 결과에 영향 없음
계정 생성 시 “개인용” / “비즈니스 관리용” 중 무엇을 선택해도 계정 기능·종류는 완전히 동일함을 확인함
안내 메일 자동 발송은 채택하지 않음
Gmail 주소로 발송 시 “보낸사람”이 그대로 노출돼 신뢰도 문제 우려 → 안내 메일은 계속 회사 공식 계정으로 발송하고, GAS는 데이터 처리만 전담
07 — 리스크와 완화

담당자 교체 시 무엇이 남는가

GAS는 배포 계정에 실행이 묶임
GitHub Pages와 달리, 배포한 계정이 비활성화되면 실제로 스크립트가 멈출 수 있음
완화: 특정 개인이 아닌 이 프로젝트 전용 구글 계정을 만들어 소유·배포
회사 공식 IT 자산 밖의 계정
현재 사내 메일(@lxintl.co.kr)은 Google Workspace가 아닌 것으로 확인됨
완화: 전용 계정 생성을 사전에 보고·문서화해 회사 공식 자산으로 관리
