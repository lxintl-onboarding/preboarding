# UX/UI 개선 계획

리서치 기반으로 도출한 개선안입니다. 우선순위 순으로 정리했으며,
각 항목의 **근거**를 함께 적었습니다. 되돌리기 전에 근거를 읽어주세요.

대상 파일: `index.html` (유일한 운영 버전)

**Phase 1은 전체 반영 완료되었습니다.** 아래 1-1~1-3은 실제로 적용된 내용의
기록으로 남겨두고, 근거가 필요할 때 참고하세요. 현재 다음 우선순위는 Phase 2이며,
최신 진행 상황은 `docs/BACKLOG.md`를 함께 확인하세요.

## Phase 1 — 레이아웃·비주얼 (완료)

### 1-1. 네비게이션을 하단 고정으로 이동 ★핵심

**현재**: 3개 탭이 헤더 바로 아래(상단)에 위치.
**변경**: 화면 하단에 `position: fixed`로 고정.

**근거**
- 사용자의 49%가 엄지만으로 앱을 조작. 엄지 도달 범위가 배치의 핵심 기준
- 6.7인치 이상 대형 화면에서 상단은 한 손으로 물리적 도달 곤란 ("Reachability Gap")
- 하단 탭은 3~5개 섹션 앱에 최적. Apple HIG / Google Material 모두 권장
- LX 앱은 4개 탭(구현 후 "입사 정보"가 "입사 준비"에서 분리되어 3→4개로 늘어남) → 여전히 이 범위에 부합
- 입사자는 첫날 전까지 여러 번 재방문해 체크리스트를 확인 → 전환 비용 최소화가 중요

**구현 요건**
```css
.tabs {
  position: fixed; bottom: 0;
  left: 50%; transform: translateX(-50%);
  width: 100%; max-width: 480px;
  background: rgba(255,255,255,.92);
  backdrop-filter: saturate(180%) blur(12px);
  border-top: 1px solid var(--line);
  padding-bottom: env(safe-area-inset-bottom, 0px);  /* iOS 노치 대응 */
  z-index: 40;
}
.sheet { padding-bottom: calc(72px + env(safe-area-inset-bottom, 0px)); }
```

- 마크업을 `<div class="sheet">` 밖, `<div class="app">` 안 마지막으로 이동
- 터치 타깃 최소 48px 확보 (`min-height: 56px`)
- 활성 탭 표시: 하단 border 대신 **아이콘 위쪽 짧은 인디케이터**
- 기존 `position: sticky; top: 0` 제거
- 탭 전환 시 `window.scrollTo({top:0})` 유지

**주의**: `.sheet`의 `border-radius: 22px 22px 0 0`와 `margin-top: -14px`는
상단 탭 전제의 스타일. 하단 이동 시 제거 또는 재조정 필요.

### 1-2. 이모지 → SVG line-icon 교체

**현재**: 탭(💌✅💡), 히어로 뱃지(🏢), 축하 모달(🎉), 회사 소개(✨),
Do&Don't(☕), 서류 안내(📌), 푸터(😊) 등에 이모지 사용.

**문제**
- 플랫폼별 렌더링 차이 (iOS/Android/Windows에서 모양이 다름)
- AI 생성물의 전형적 신호. 톤이 유치해짐
- 브랜드 컬러와 무관한 색이 화면에 섞임

**변경**
- **탭 4개**: SVG line-icon (stroke-width 1.6, `stroke="currentColor"`)
  - 환영 인사 → 봉투(envelope)
  - 입사 정보 → 위치 핀
  - 입사 준비 → 클립보드 체크
  - 참고 정보 → 전구(lightbulb)
- **나머지 이모지**: 제거하거나 SVG로 대체
- 예외: 축하 모달의 🎉는 유지 검토 가능 (일회성 정서적 순간)

아이콘은 인라인 SVG로. 외부 아이콘 라이브러리 도입 금지 (단일 파일 배포 원칙).

**반영 결과**: 4개 탭 모두 SVG 아이콘 적용 완료, 이모지는 전부 제거되었습니다.

### 1-3. 타이포 · 여백 · 그림자 정리

**타이포 위계 강화**
- 현재는 크기 차이로만 위계를 만듦 → 대비를 과감하게 벌릴 것
- 제목: 굵게(800) + `letter-spacing: -.02em`
- 본문: 400~500, 차분하게
- 부가정보: 확실히 작고 흐리게
- 폰트는 Pretendard 하나, weight만 400/600/800으로 분리

**색 절제**
- **한 화면에 강조색은 하나.** LX Red가 주인공, 나머지는 웜그레이 무채색
- Gold는 아주 가끔 포인트로만 (현재 과다 사용 지점 점검)

**여백 확대**
- 카드 안팎 padding/margin 증량. "비었나?" 싶을 때가 적당
- 섹션 간 구분은 선(border)보다 여백으로

**그림자 절제**
- 모든 카드에 그림자를 주면 촌스러움
- `.card`(인터랙티브)에만 유지, `.card-info`(읽기 전용)는 이미 제거됨 — 이 원칙 확장

## Phase 2 — 콘텐츠

### 2-1. 환영 인사 탭 보강 (부분 반영됨)

"웰컴 노트"가 "동료들의 환영 인사"(Welcome Note) 섹션으로 이름은 바뀌었지만,
내용은 여전히 "여러분의 도전과 열정이…" 같은 일반론이고 서명도 "LX인터내셔널
임직원 일동"입니다. 아래 4개 항목은 전부 아직 미반영.

**추가할 섹션 (우선순위순)**

1. **팀/동료의 한마디** — 동료가 보내는 메시지는 '회사'가 보내는 것과 다르게 와닿음.
   짧은 메시지라도 소속감(belonging)을 크게 높임. 현업 부서 환영 문구 2~3개.

2. **첫 주 로드맵** — "첫날 오리엔테이션 → 첫 주 팀 소개·시스템 셋업 → 첫 달 …"
   언제 무엇을 기대할지 명확히 해 첫날 불안을 줄임.

3. **FAQ 미니 섹션** — "점심은?", "주차 가능?", "첫날 복장?" 등 실질적 궁금증.

4. **CEO 메시지 구체화** — 일반론 대신 LX 실제 사업(무역·자원·물류)과 연결.
   대표이사 실명·직함 명시.

**설계 원칙**: 한 번에 다 쏟아붓지 말 것(drip). 물류 → 맥락·문화 → 동료 소개 순.

### 2-2. 참고 정보 탭 보강 (부분 반영됨)

**반영된 것**: 탭 이름이 "참고 정보"로 바뀌었고(제안했던 "미리 알아두기"는 아니지만
같은 취지), "LX인터내셔널 문화" 필터 그룹에 인재상·인재 육성 체계·신규 입사자 대상
프로그램·조직문화 프로그램·출근룩 가이드 5개 아코디언이 LX 고유 콘텐츠로 추가되었습니다.
기존 "기본 매너·에티켓" 그룹(첫날 가이드/커뮤니케이션/회의·보고/동료관계/보안)은 유지.

**아직 안 된 것** — 아래 항목은 여전히 어디에도 없어 신입이 헤맬 수 있는 정보입니다:
- 사내 호칭 문화 (님/책임/선임 등 직급 체계)
- 자주 쓰는 사내 용어·약어
- 조직 구조 개요
- 주요 복지 제도 (식대, 경조사, 휴가 신청 방법)

### 2-3. 퀴즈 3문항 (미반영)

참고 정보 탭 하단. 읽고 → 확인하는 학습 루프.
Appical이 프리보딩 핵심 축으로 꼽는 "인터랙티브 콘텐츠".

## Phase 3 — 인터랙션·구조

### 3-1. 재방문 시 진입 탭 분기

현재는 항상 "환영 인사"가 먼저 열림. 두 번째 방문부터는 체크리스트를 보고 싶을 것.

**옵션 A**: localStorage로 방문 이력 판단 → 재방문 시 "입사 준비"로 진입
**옵션 B**: 환영 탭 하단에 "입사 준비 시작하기 →" 큰 CTA 버튼

### 3-2. 입사 준비 탭 스크롤 길이 관리 (다른 방식으로 해결됨)

이 항목을 제안할 당시엔 첫 출근 안내 + 진행률 + To-Do 7개(서류 9종 중첩) +
배송지 폼이 한 탭에 몰려 있었는데, 이후 "첫 출근 안내"가 별도의 "입사 정보"
탭으로 분리되고 To-Do도 5개로 정리되면서 스크롤 길이 문제 자체가 해소됐습니다.
제안했던 "배송지 폼 접이식" 방식은 아니지만, 목표(길이 관리)는 달성된 상태라
추가 작업 불필요.

### 3-3. 마이크로 인터랙션

- 체크 시 부드러운 애니메이션 (현재 있음, 유지)
- 탭 전환 페이드 (현재 있음, 유지)
- 과하지 않게. `prefers-reduced-motion` 대응 필수
